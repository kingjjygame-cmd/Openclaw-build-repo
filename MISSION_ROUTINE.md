# 미션 스테이트먼트 실행 루틴(수동 운영)

목표: `테크·앱개발 역량으로 실행 자동화를 구축하고, 재무·투자·부동산 의사결정을 데이터로 정밀하게 관리해, 12년차 엔지니어로서 가족과 함께 안정적이고 지속 가능한 금전적 자유를 달성한다.`

## 매일 루틴

### 1) 아침(07:30~08:00) — 오늘 시작 브리핑
- **오늘 완료할 핵심 3개 미션만 확인**
  1. 기술/앱 자동화 작업
  2. 재무/투자 점검 항목
  3. Hebbal ELT 관련 조치
- Mission Control에서 상태 확인: `진행중/완료/지연`을 점검
- 아침 메시지 톤: `지난 작업에서 성과가 있었고, 오늘은 OOO로 시작` 형태의 간단 응원 리포트

### 2) 점심/오후(14:30~15:00) — 미션 점검 1차
- **짧은 실행 액션 1개 추가** (15~30분)
- 기술: CI/스크립트/개인 앱에서 바로 실행 가능한 단위 작업 1개
- 재무/부동산: 숫자/리스트 업데이트 1개 항목만 확정
- 미션 컨트롤에 진행률 반영 + 체크리스트 토글

### 3) 밤(21:30~22:00) — 데일리 리플렉션
- 오늘 한 줄 성과 기록(`오늘 한 줄`)
- **미션 컨트롤에서 백업 실행** 또는 스냅샷 상태 확인
- 내일 우선순위 2개로 압축 정리
- “완료한 일”만 먼저 보고하고, 미완료는 다음 액션 1개로 줄임

## 핵심 규칙
- 매 세션 시작 시 3가지 질문만 고정:
  1. 미션 스테이트먼트에 가장 근접한 오늘의 성과는?
  2. 가장 큰 병목은 무엇인가?
  3. 내일 첫 30분에 할 일은 무엇인가?
- 과도한 확장보다 **1개 미션, 1개 실행 단위**로 마무리한다.

## 스케줄 자동화(선택)
원하면 내가 cron으로 아래를 자동 실행하도록 구성할게:
- `npm run dev` 상태 체크는 매일 1회
- `mission-control` 백업/복원 로그 점검은 매일 밤
- 핵심 체크리스트 알림은 요약 텍스트로 기록
