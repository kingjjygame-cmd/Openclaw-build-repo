# Mission Control (Next.js 운영용) 초기 설정/실행 가이드

준영님 전용 로컬 미션 컨트롤 대시보드입니다.

## 1) 환경 재설정(초기화) 실행 순서

### 0. 디렉터리 이동
```bash
cd /home/kingjjy_game/.openclaw/workspace/mission-control
```

### 1. 기존 실행 정리 (선택)
```bash
# 실행 중이면 터미널에서 Ctrl+C
# 또는 기존 프로세스 종료 후
```

### 2. 의존성 재설치 (클린 재설치)
```bash
rm -rf node_modules package-lock.json
npm install
```

### 3. 타입/빌드 검증
```bash
npm run build
```

### 4. 로컬 실행
```bash
npm run dev
```

- 브라우저: http://localhost:3000

### 5. 운영 모드 실행(운영 테스트 시)
```bash
npm run build
npm run start
```
`start`는 `next start`로 기본 **3000번 포트**에서 정적 빌드 결과를 서빙합니다.

## 2) 데이터 저장/백업 (단일 사용자 모드)
- 기본 데이터 저장: 브라우저 `localStorage`
  - 실 데이터: `mission-control:data:v1`
  - 스냅샷 메타: `mission-control:snapshots:v1`
- 자동 스냅샷: 앱 실행/변경 시 당일 1회(`YYYY-MM-DD`)로 자동 생성
- 최대 보관: 최근 14개
- 초기화 버튼: **로컬 데이터 초기화** 버튼으로 상태 + 스냅샷 삭제

## 3) 운영용 기능 사용법
- 미션 등록: 상단 폼에서 제목/목표/마감일/우선순위 입력 후 등록
- 상태 전환: 각 카드의 드롭다운(진행중/완료/지연/대기)
- 체크리스트: 항목 체크 시 진행률 자동 반영
- 진행률 조정: 슬라이더로 수동 조정 가능(체크리스트 계산과 별도 반영)
- 오늘 메모: 오늘 한 줄 입력 후 `메모 저장`
- 백업 파일 생성: **현재 상태 백업(.json)**
- 백업 복원: **백업 파일 복원**으로 .json 파일로 이전 상태 불러오기
- 최근 스냅샷 복원: 최근 자동/수동 스냅샷에서 즉시 복구
- 데이터 손상 대비: 복구 후 브라우저 새로고침해서 영속 저장 여부 확인

## 4) 운영 확장 포인트
